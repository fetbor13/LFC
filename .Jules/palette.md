## 2024-05-20 - Missing ARIA Roles and Labels in Custom Modals
**Learning:** This application uses custom modal implementations (`div.modal`, `div.profile-modal`) rather than native `<dialog>` elements. These custom implementations lack `role="dialog"`, `aria-modal="true"`, and `aria-labelledby` attributes by default, making them inaccessible to screen readers. Furthermore, inputs inside these modals often rely solely on `placeholder` text without an explicit `<label>` or `aria-label`, hiding their purpose from assistive technologies.
**Action:** When working on UI components in this app, always verify that custom modals implement proper ARIA roles (`dialog`, `aria-modal`) and connect title IDs using `aria-labelledby`. Ensure all form inputs have a clear `aria-label` if a visible `<label>` is not present.
